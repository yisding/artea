# Artea PyPI pull-through constraints (R3) — devpi-constrained format.
#
# Lives in the configured Gitea policy repo. Edit via PR; on push,
# policy-sync applies this file as the `constraints` property of the devpi
# index `root/constrained` (the PyPI mirror the gateway falls back to).
#
# Syntax: one rule per line, pip-constraints-like (PEP 440 specifiers).
# A package matching a line is limited to the versions allowed by it;
# packages with no matching line are unrestricted. Lines starting with `#`
# and blank lines are ignored.
#
#   urllib3<2              # only versions < 2 are visible to pip
#   requests ==2.31.0      # pin a package to exactly one version
#   pyyaml>=5.4,<7         # compound specifiers are allowed
#   pkgname==0             # effectively block a package (no version 0 exists)
#   *                      # default-deny: hide everything NOT listed above
#
# Only pull-through of public PyPI packages is affected. Private packages are
# served by Gitea and never pass through devpi.
#
# Public upstream recency gates live in upstream-policy.yaml and apply across
# formats.
#
# Default policy: no constraints (everything on PyPI allowed).
